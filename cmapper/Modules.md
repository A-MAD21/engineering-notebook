# CMapper Modules

CMapper is built around independent modules responsible for collecting specific types of infrastructure information.

## Design Principles

- Independent execution
- Easy extensibility
- Vendor-specific collection
- Normalized output
- Queue-driven scheduling

## Example Modules

- SSH Discovery
- CDP Discovery
- LLDP Discovery
- Ping Discovery
- Ubiquiti Integration
- Service Discovery
- Database Synchronization

Additional modules can be added without redesigning the platform.
